# Учебный комплект (3 задания)

Репозиторий содержит три независимых папки с заданиями:
- `task1-kafka-playground` — локальный Kafka стенд через Docker Compose: создание топика, отправка/чтение сообщений, анализ метаданных.
- `task2-go-microservice` — Go REST API с CRUD, rate limiting, метриками Prometheus и интеграцией MinIO.
- `task3-go-k8s-analytics` — Go сервис с потоковой аналитикой (rolling average + z-score), Redis-кэшем и манифестами Kubernetes (HPA).

Все команды приведены под **PowerShell 7.5.4** (под Windows 11). В онлайн-лабораториях Killercoda/аналогах для заданий 2–3 используйте тот же синтаксис: отличий почти нет, кроме обращения к переменным окружения (`$env:` в PowerShell).

## Что делать пошагово
1. Прочитать README внутри каждой папки.
2. Выполнить чек-листы.
3. Собрать скриншеты/результаты и вставить в соответствующие текстовые шаблоны `docs/report-template.md`, затем экспортировать в .docx/.pdf (бинарные файлы не храним в репозитории).
4. Для задания 1 используйте локальный Docker Compose (см. `task1-kafka-playground/docker-compose.yml`) вместо онлайн-лаборатории.
5. Для заданий 2 и 3 (Go-проекты) при необходимости собрать Docker-образы и прогнать нагрузочные тесты.

## Общие чек-листы
- [ ] `task1-kafka-playground`: 4 скриншота (topics, create, consumer, kcat -L); ответы про партиции и метаданные.
- [ ] `task2-go-microservice`: CRUD работает; rate limiting выдаёт 429 при перегрузке; `/metrics` отображает данные; wrk > 1000 rps; MinIO upload успех; отчет заполнен.
- [ ] `task3-go-k8s-analytics`: локальный запуск; Redis подключен; `/metrics` собирается Prometheus; нагрузка >1000 rps; HPA масштабируется; отчет + презентация готовы.

## Полезные подсказки
- Если команда «зависла» в онлайн-лабе, жмите `Ctrl+C` и проверяйте состояние (например, `kafka-topics.sh --describe`).
- Все переменные окружения в PowerShell: `$Env:VAR`. В Bash: `$VAR`.
- Для запуска Docker Compose в PowerShell используйте `docker compose up` (без дефиса).

## Состав работы
- Код и конфигурации по каждому заданию.
- Текстовые шаблоны отчетов (`docs/report-template.md`) для дальнейшего экспорта в **.docx/.pdf** (бинарные файлы не хранятся в репозитории).
- Контейнеризация (Docker/Docker Compose) для Go-проектов.
- Kubernetes-манифесты (Deployment, Service, HPA, ConfigMap) для задания 3.
